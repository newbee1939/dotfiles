# Commit（コミット作成）

変更を意味のある単位でコミットする。**コミットメッセージは日本語**。

## 手順
1. `git status` と `git diff`（ステージ済みは `git diff --staged`）で変更内容を把握する。
2. 関連しない変更が混ざっていれば、論理単位に分けて複数コミットにする。
3. 意図したファイルだけをステージし、日本語でメッセージを書く:
   - 1 行目: 何をしたかを簡潔に（例: 「〜を追加」「〜を修正」）
   - 空行
   - 本文: **なぜそうしたか**（背景・理由）。What だけでなく Why を書く。
4. コミット前に、対象ファイルとメッセージをユーザーに見せて確認する。
5. リポジトリ既存コミットの粒度・文体に合わせる。

## 注意
- `git add -A` で無関係な変更を巻き込まない。
- 機密情報（鍵・トークン・`.env`）が差分に含まれていないか確認する。
- main へ直接コミットしない運用なら、先にブランチを切る。
