# Start Task（タスク開始）

新しい作業・チケットに着手するときの起点。
**全てのタスクが実装（コード変更）を伴うわけではない**ので、まず性質を見極めてから進める。

## 1. タスクの性質を見極める
何のタスクか（チケット名・概要・ゴール）を確認し、次を判断する:
- **実装が必要か？**（コードを書く・ファイルを変更する）
- それとも調査・設計・運用・議論・ドキュメントなど、**コード変更を伴わない**タスクか？

→ 実装が必要なら **2A**、不要なら **2B** へ。

## 2A. 実装タスク：git を準備する
1. 未コミットの変更が無いか `git status` で確認する。
2. main を最新化する:
   ```
   git checkout main
   git pull origin main
   ```
3. 作業ブランチを切る:
   ```
   git checkout -b <prefix>/<topic>
   ```
   例: `feat/add-aerospace`, `fix/login-redirect`。prefix は feat/fix/chore/docs など。ブランチ名は内容が分かる短い英語にする。
4. 並行作業なら worktree を使う:
   ```
   git worktree add ../<repo>-<topic> -b <prefix>/<topic>
   ```
   - ディレクトリ名は **ブランチ名の `prefix/` を外してハイフン区切り**にする（スラッシュをそのまま使うと中間ディレクトリができて扱いにくい）。
   - 作業ツリーはリポジトリの外（親ディレクトリ直下）に置く。

### 完了時の後片付け（worktree を使った場合）
1. メインの作業ツリーに戻る（worktree の中にいると削除できない）。
2. worktree を削除する: `git worktree remove ../<repo>-<topic>`（未コミットの変更があると失敗。意図的に捨てるなら `--force`）。
3. マージ済みのブランチを削除する: `git branch -d <prefix>/<topic>`（未マージで強制削除する場合のみ `-D`）。

## 2B. 非実装タスク：アシスタントとして伴走する
ブランチは切らない。タスクを一緒に前へ進める:
1. ゴールと完了条件（何をもって終わりとするか）を言語化する。
2. タスクを小さなステップに分解する。
3. 必要な調査・情報収集・整理を手伝い、成果（メモ・結論・次の一手）をまとめる。
4. 進める中で実装が必要だと分かったら **2A** に切り替える。
