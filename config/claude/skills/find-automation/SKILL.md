# Find Automation（自動化の発見）

今のセッション（または指定された作業ログ）を振り返り、**自動化・効率化できる余地**を洗い出す。

## 手順
1. 直近の作業を振り返り、次のパターンを探す:
   - 繰り返している手作業・定型コマンド列
   - 毎回同じ説明・チェックリストを Claude に貼っている箇所
   - 手順が決まっているのに毎回手で実行している操作
2. 見つかった候補ごとに、最適な自動化手段を判定する:
   - **skill 化**: 手順・チェックリストが定まった作業（`config/claude/skills/`）
   - **シェル関数 / エイリアス**: 純粋な CLI 操作の短縮（`shell/.zshrc.extra.zsh`）
   - **hook / settings.json**: 「〜したら毎回」の自動実行
   - **スクリプト**: 複数ステップのまとまった処理（`scripts/`）
3. 各候補を「効果 × 実装コスト」で並べ、効果が大きく軽いものから提案する。
4. ユーザーが選んだものを実際に作る。

## 原則
- 1〜2 回しかやらない作業は自動化しない（コストに見合わない）。
- まず最も軽い手段を提案する（新規ファイルを増やさない案を優先）。
- 既存の skill / スクリプトと重複しないか確認する。
